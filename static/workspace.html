Connect AWS
Cryptiq reads ACM certificates, KMS keys, Route53 records, and EC2 hosts — read-only, nothing is ever modified. Keys are encrypted at rest and never appear in logs.
1Create an IAM user in your AWS account named cryptiq-scanner.
2Attach this inline policy exactly as written below.
3Generate an access key for that user and drop both values in here.
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Action": [
      "acm:ListCertificates",
      "acm:DescribeCertificate",
      "kms:ListKeys",
      "kms:DescribeKey",
      "ec2:DescribeInstances",
      "route53:ListHostedZones",
      "route53:ListResourceRecordSets",
      "elasticloadbalancing:DescribeListeners",
      "elasticloadbalancing:DescribeLoadBalancers"
    ],
    "Resource": "*"
  }]
}
